=====================================================
CACHING STRATEGY
=====================================================

READ OPERATIONS (@Cacheable):
- First call: Queries database, stores in cache
- Subsequent calls: Returns from cache (fast)
- Cache key based on method parameters

WRITE OPERATIONS:
- CREATE: Evicts list caches (products, categories, users)
- UPDATE: Updates specific cache entry + evicts list caches
- DELETE: Evicts all related caches

CACHE EVICTION:
- Single item operations: Target specific cache entries
- List operations: Clear entire cache to maintain consistency
- Multi-cache operations: Use @Caching for coordinated eviction

=====================================================
CACHE EVICTION EXAMPLES
=====================================================

1. CREATE PRODUCT:
   - Evicts: products cache (all entries)
   - Reason: List changed, need to refresh on next request

2. UPDATE PRODUCT:
   - Updates: product cache (specific ID)
   - Evicts: products cache (all entries)
   - Reason: Single item updated, lists need refresh

3. DELETE PRODUCT:
   - Evicts: product cache (specific ID) + products cache (all)
   - Reason: Item removed, clean up all references

4. UPDATE USER (email changed):
   - Updates: user cache (by ID) + userByEmail cache
   - Evicts: users cache (all entries)
   - Reason: Multiple lookups need sync

=====================================================
CACHE MANAGEMENT
=====================================================

Current Implementation:
- In-memory concurrent map cache
- No size limits (production should add limits)
- No TTL (Time To Live) - cache lives until evicted
- Thread-safe concurrent access

Production Considerations:
- Add cache size limits
- Add time-based eviction (TTL)
- Consider distributed cache (Redis, Hazelcast)
- Add cache statistics and metrics

To add TTL/Size limits, upgrade to Caffeine:
1. Add dependency: com.github.ben-manes.caffeine:caffeine
2. Use CaffeineCacheManager instead of SimpleCacheManager
3. Configure expireAfterWrite, maximumSize
